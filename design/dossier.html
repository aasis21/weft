W WEFT 01 — The Brief
The problem

Walk away
from the desk.

Agent runs are long. Repo-wide refactors, /fleet commands, multi-step plans — they don't need you watching. Until, suddenly, they do: a file write to approve, a test to rerun, a wrong turn to stop.

Weft is the encrypted viewport that lets you leave the terminal and still hold the controls — without devtunnels, inbound firewall rules, or a background daemon eating your RAM.

Operating principles
⏻  Ephemeral
The channel lives exactly as long as your gh copilot session. Close the terminal and it vanishes — no zombies, no background service.
🔒  Air-gapped
A fresh AES-256-GCM key is derived on-device from a scanned QR. The relay only ever carries ciphertext.
⌘  Native
A true CLI extension on the agent's RPC loop — not a scraper, not a wrapper. It sees events as they happen.

W WEFT 02 — System Architecture
How it connects

Three nodes,
one encrypted seam.

Your phone never talks to the agent directly. It speaks to a volatile Supabase channel, which routes opaque envelopes to the extension riding inside your terminal. The relay is a conduit, not a store.

The Conduit

The Host
CLI Extension
Hooks the agent's JSON-RPC loop. Forwards tokens, holds the loop on tool requests.
Lifecycle
The extension is a child of gh copilot. It opens the channel on launch, prints a pairing QR, and dies when you type /exit. The phone reports “Session ended.”
Trust boundary
Keys are generated at the endpoints and never transit the relay. Intercepted traffic is ciphertext only. No inbound ports, no tunnels, no browser auth on the phone.

W WEFT 04 — Multi-Session
Many terminals, one app

Every session,
kept warm.

Join more than one gh copilot session and Weft keeps them all connected at once — each with its own live timeline held in memory. Switch in a tap; scrollback is preserved.

Joined sessions persist to local storage and reconnect on launch. The drawer surfaces each one's status, message count, age and working directory — and flags any that are waiting on you.

Per-session state
● Connecting→ ● Live→ ● Quiet→ ● Ended
Concurrent
Each session keeps its own SecureChannel and reducer — switching never drops a stream.
Persistent
Stored under weft.sessions.v1, with migration from the legacy single-pairing key.
Self-healing
A heartbeat watchdog eases live → quiet after silence, instead of falsely declaring the session dead.
